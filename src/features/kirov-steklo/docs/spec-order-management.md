# Техническое задание на разработку производственной системы управления заказами

## 1. Информация о проекте

**Заказчик:** Марат Гайнулович (производственная компания)

**Исполнитель:** Компания Проксима Айти (технический директор веб-отдела: Павел)

**Срок предоставления коммерческого предложения:** до 23.11.2025

## 2. Описание бизнеса заказчика

Компания занимается производством зеркал и стекол под индивидуальные размеры с различными параметрами:

- Различные формы и размеры
- Разные виды обработки
- С подсветкой и без подсветки
- Дополнительные опции (сверление, крепления и др.)

**Сегменты клиентов:**

- B2B: мебельные компании, дизайнерские бюро, архитектурные и строительные организации
- B2C: частные лица через сайт

## 3. Текущая ситуация и проблемы

- Ведение учета в электронных таблицах
- Отсутствие единой IT-системы для управления производством
- Нет прозрачности производственного процесса для руководства
- Множество разрозненных каналов приема заявок
- Используется Битрикс24, но система плохо справляется с отслеживанием статусов заявок и их прохождением через производственный цикл

## 4. Цели и задачи проекта

**Главная цель:** Создать единую IT-систему для полного цикла управления заказами от приема заявки до выдачи готовой продукции с устранением электронных таблиц.

**Основные задачи:**

- Сбор и формирование заявок из всех каналов
- Формирование заявки для производства с созданием производственной карточки
- Интеграция с CRM (Битрикс24)
- Проведение заявки через все этапы производства
- Отслеживание статуса
- Информирование менеджера и клиента
- Прозрачность для руководителя

## 5. Описание производственного процесса

**Этапы:**

1. Формирование заявки менеджером, создание ТЗ
2. Оплата (банковское приложение, счет юр. лицу)
3. Производство: материал, обработка, дополнительные операции (сверление, крепления, подсветка)
4. Упаковка, фиксация готовности
5. Склад/Выдача
6. Логистика (опционально)
7. Монтаж (опционально)

## 6. Функциональные требования

**Менеджеры:**

- Создание/редактирование заявок
- Формирование ТЗ
- Просмотр статусов и истории
- Информирование клиентов

**Производство:**

- Получение карточек с ТЗ
- Обновление статусов
- Просмотр очереди заказов
- Фиксация завершения этапа

**Руководитель:**

- Мониторинг процессов
- Аналитика и отчетность
- Контроль загрузки

**B2B клиенты:**

- (Обсуждается) Личный кабинет
- Информирование (SMS, Telegram, WhatsApp)
- История заказов, отслеживание

**B2C клиенты:**

- Уведомления о статусе заказа
- Информация о готовности
